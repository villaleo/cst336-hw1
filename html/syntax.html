About the 'C'-ntax (Syntax)

C++'s syntax is very similar to C's syntax since C++ is a successor of C. In fact, most C programs can be compiled using a C++ compiler. If a programmer wanted to, they could write C code in a C++ file and compile it as C++ code. The code will execute just fine. Though, doing this is frowned upon by many C++ programmers as they urge other programmers to stick with an unwritten standard. Most C++ programmers will see a C++ program using C-style semantics or overall style and will immediately call out the author to use the C++ STL or a C++ equivalent. I don't entirely understand why that is, but it seems that many C++ programmers are trying to steer away from C++ being derived from C and calling anything that's C-style bad.

As of C++20, there are 70+ reserved keywords. It's important to know these, since you cannot have a variable or function with the same name as any of these tokens.
